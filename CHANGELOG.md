# Changelog

所有重要变更都会记录在这里。

## [0.4.3] - 2026-05-16

### Added

- 新增 `notify_no_updates` 配置，可在定时检查无新推文或首次记录账号时发送检查摘要。
- 新增管理员诊断命令 `/推文状态`、`/推文检查`，支持查看调度器状态并手动触发一次定时检查。

### Changed

- 定时检查改为输出更清晰的 info 级别日志，包含触发原因、关注账号、推送目标、无更新、初始化和失败统计。
- 调度器增加启动兜底，在插件初始化和 AstrBot 加载完成时都会尝试确保任务运行。

## [0.4.2] - 2026-05-15

### Added

- 增加翻译链路日志，可看到翻译是否开启、每条推文是否触发、跳过原因、provider 来源和模型返回结果。
- 翻译 provider 增加兜底逻辑：未配置模型且当前会话无默认模型时，尝试使用 AstrBot 已配置 provider 列表中的第一个。

## [0.4.1] - 2026-05-15

### Changed

- 简化翻译提示词配置：将原来的系统提示词和用户提示词模板合并为一个 `translate_prompt`。
- 翻译调用改为只向 `llm_generate` 传入一份 prompt，配置界面更清爽。

### Docs

- README 增加访问计数图片。

## [0.4.0] - 2026-05-15

### Added

- 新增非中文推文翻译功能，可调用 AstrBot 已配置的大模型生成简体中文译文。
- 新增 `translation_provider_id`，支持通过 WebUI 选择已有模型 provider。
- 新增 `translator.py`，将语言检测和翻译调用从抓取、发送逻辑中拆出。
- 手动查询和定时推送均支持在发送前追加译文。

## [0.3.0] - 2026-05-15

### Added

- 新增定时检查与推送功能。
- 支持关注多个用户并按间隔或每日定点检查新推文。
- 支持 `push_targets` 配置多个推送目标。
- 首次启用账号时只记录当前已见推文 ID，不推送历史内容。
- 新增 KV 已见状态存储，每个账号保留最近 100 个已见推文 ID。

## [0.2.0] - 2026-05-15

### Added

- 新增推文媒体解析与下载。
- 将 Nitter 推文链接转换为 `x.com/<user>/status/<id>` 后传入 xdown 解析。
- 支持下载图片、视频和 GIF，并作为合并转发消息段发送。
- 媒体解析或下载失败时保留文本和链接发送。

## [0.1.0] - 2026-05-15

### Added

- 初始版本。
- 支持通过 Nitter RSS 获取指定用户最近公开推文。
- 支持 `/推文`、`/tweets` 等命令别名。
- 支持以聊天记录/合并转发形式发送推文。
