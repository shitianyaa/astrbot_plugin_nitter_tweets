# Changelog

所有重要变更都会记录在这里。

## [0.6.3] - 2026-06-07

### Fixed

- 修复飞书/Lark 平台可能只发送图片、正文文本丢失的问题；飞书现在优先通过原生 `text` 消息发送正文，再发送图片/视频附件。
- 修复飞书/Lark 定时推送和合并推送目标通过 UMO 主动发送时没有走飞书原生文本发送路径的问题。

### Changed

- README、配置提示和插件元数据补充飞书/Lark 推送目标示例与平台发送差异说明。
- 插件元数据按 AstrBot `support_platforms` 字段声明实际适配器；定时推送发送状态异常只记录短日志，不再向推送目标发送额外提示消息。

## [0.6.2] - 2026-06-06

### Fixed

- 修复 OneBot/QQ 合并转发已送达但回包超时或网络状态不确定时，插件误判失败并发送去视频版或纯文本 fallback，导致同一轮重复推送的问题。
- 修复手动 `/推文` 在同类不确定送达异常下仍可能触发降级重发的问题。
- 修复后台定时检查与手动 `/推文检查` 并发时可能在已见 ID 写入前重复推送同一批新推文的问题。

### Changed

- 定时推送遇到不确定送达时会按可能已送达计入成功，跳过降级重试。
- 插件行为展示中的指令说明改为中文，并为 AstrBot 加载完成钩子补充描述。

## [0.6.1] - 2026-06-05

### Fixed

- 修复 Telegram、微信等非 OneBot 平台因合并转发节点不受支持导致不推送的问题；非 QQ 平台会自动使用普通消息链。
- 修复手动 `/推文` 发送路径在部分平台无支持消息段时没有明确失败提示的问题。
- 修复 Nitter RSS 将 YouTube 链接改写为 `piped.video` 后在正文或译文中重复出现的问题；现在仅将 Piped 链接还原为 `youtu.be`，其他外链保留在原文位置。
- 修复 URL 提取误识别邮箱域名的问题。
- 修复 Linux 环境下本地媒体文件 URI 多出斜杠，导致 OneBot 可能收到 `//AstrBot/...` 路径的问题。
- 修复视频推文的封面图可能被当作普通图片一起发送的问题。

### Added

- 新增 `send_image_attachments` 和 `send_video_attachments` 配置；配置界面改为只展示“发送图片附件”和“发送视频/GIF 附件”两个开关。
- `send_video_attachments` 默认关闭；关闭时保留原帖链接并提示打开原文查看。

### Changed

- 定时推送的合并转发仅 OneBot v11/`aiocqhttp` 使用；其他平台发送普通消息链。
- 翻译输入改为去除 URL 后的正文，避免译文重复输出链接。
- `push_targets` 配置说明改为推荐使用 `/sid` 获取完整 UMO；配置界面不再展示 `platform_id`。
- 精简 README，并补充多镜像重试、手动检查、多目标推送、平台发送差异和 AI 协助说明。
- 移除配置界面中的 `download_media`、`download_images`、`download_videos`，升级后以新的图片/视频发送开关为准。

## [0.6.0] - 2026-05-23

### Added

- 新增 `media_cache_retention_days` 配置，默认自动清理超过 3 天的媒体缓存。
- 新增 `vision_max_images` 配置（替代原 `vision_first_image_only`），支持每条推文识别多张图片，识图并发执行（asyncio.gather）。
- 发送合并转发消息失败时，自动去掉视频重试一次；视频位置替换为文本链接。
- 媒体类型检测增加 URL 扩展名兜底，解决 xdown 返回链接文字不含关键词时无法识别类型的问题。

### Changed

- 重构代码结构：`models.py` 合入 `utils.py`，`translator.py` 合入 `enricher.py`，`nitter_client.py` 合入 `media.py`，文件从 9 个精简为 6 个。
- 重写 enricher 模块，识图与评论的 provider 解析统一为单一 `_resolve_provider` 方法。
- 统一日志前缀为 `[NitterTweets]`。

## [0.5.0] - 2026-05-21

### Added

- 新增 AI 评论功能，可按概率调用文本大模型，为推文追加简短中文点评。
- 新增 AI 识图功能，可按概率调用视觉模型识别推文第一张图片，并把描述追加到推文节点。
- 新增 `comment_provider_id`、`vision_provider_id`、概率和提示词配置，默认关闭以避免升级后自动产生模型调用费用。
- 新增管理员命令 `/推文订阅列表`，用于查看当前有效订阅作者、重复项和无效项统计。

### Changed

- 定时调度默认不再在插件重启后立即检查新推文；如需旧行为可开启 `check_on_startup`。
- 手动查询和定时推送都会在媒体下载后执行 AI 增强；模型调用失败时保留原推文、链接和媒体发送。

## [0.4.4] - 2026-05-19

### Added

- 新增 `merge_scheduled_updates` 配置，可将一次定时检查中所有账号的新推文合并为一条聊天记录发送。
- 新增管理员命令 `/推文订阅去重`，支持规范化并去重 `watch_users`，移除重复作者和无效条目后保存配置。

### Changed

- `/推文状态` 现在会显示合并推送开关，以及订阅作者配置项、重复项和无效项统计。

## [0.4.3] - 2026-05-16

### Added

- 新增 `notify_no_updates` 配置，可在定时检查无新推文或首次记录账号时发送检查摘要。
- 新增管理员诊断命令 `/推文状态`、`/推文检查`，支持查看调度器状态并手动触发一次定时检查。

### Changed

- 定时检查改为输出更清晰的 info 级别日志，包含触发原因、关注账号、推送目标、无更新、初始化和失败统计。
- 调度器增加启动兜底，在插件初始化和 AstrBot 加载完成时都会尝试确保任务运行。

## [0.4.2] - 2026-05-15

### Added

- 增加翻译链路日志，可看到翻译是否开启、每条推文是否触发、跳过原因、provider 来源和模型返回结果。
- 翻译 provider 增加兜底逻辑：未配置模型且当前会话无默认模型时，尝试使用 AstrBot 已配置 provider 列表中的第一个。

## [0.4.1] - 2026-05-15

### Changed

- 简化翻译提示词配置：将原来的系统提示词和用户提示词模板合并为一个 `translate_prompt`。
- 翻译调用改为只向 `llm_generate` 传入一份 prompt，配置界面更清爽。

### Docs

- README 增加访问计数图片。

## [0.4.0] - 2026-05-15

### Added

- 新增非中文推文翻译功能，可调用 AstrBot 已配置的大模型生成简体中文译文。
- 新增 `translation_provider_id`，支持通过 WebUI 选择已有模型 provider。
- 新增 `translator.py`，将语言检测和翻译调用从抓取、发送逻辑中拆出。
- 手动查询和定时推送均支持在发送前追加译文。

## [0.3.0] - 2026-05-15

### Added

- 新增定时检查与推送功能。
- 支持关注多个用户并按间隔或每日定点检查新推文。
- 支持 `push_targets` 配置多个推送目标。
- 首次启用账号时只记录当前已见推文 ID，不推送历史内容。
- 新增 KV 已见状态存储，每个账号保留最近 100 个已见推文 ID。

## [0.2.0] - 2026-05-15

### Added

- 新增推文媒体解析与下载。
- 将 Nitter 推文链接转换为 `x.com/<user>/status/<id>` 后传入 xdown 解析。
- 支持下载图片、视频和 GIF，并作为合并转发消息段发送。
- 媒体解析或下载失败时保留文本和链接发送。

## [0.1.0] - 2026-05-15

### Added

- 初始版本。
- 支持通过 Nitter RSS 获取指定用户最近公开推文。
- 支持 `/推文`、`/tweets` 等命令别名。
- 支持以聊天记录/合并转发形式发送推文。
