# Changelog

所有重要变更都会记录在这里。

## [0.9.1] - 2026-06-09

### Fixed

- 媒体缓存目录从插件源码目录迁移到 AstrBot 插件数据目录，避免本地图片/视频缓存导致 Windows 重装或删除插件目录时提示文件夹被占用。
- `/推文缓存清理` 现在会同时尝试清理当前插件数据目录缓存和旧版源码目录 `cache/` 根目录下的普通缓存文件；暂存队列媒体目录仍会保留。
- 旧版 KV seen 数据迁移到 SQLite 后会自动删除，避免卸载时勾选删除持久化数据后，重装又从旧 KV 恢复旧的“已记录账号”。
- 定时检查结果会在 seen 写入后刷新“已记录账号索引”数量，并调整状态文案，避免误解为订阅账号数或推文数量。
- 手动 `/推文检查` 对全局分组改为按顶层 `push_targets` 判断当前对话归属；当前对话在全局推送目标中时，即使后台定时总开关关闭也可手动检查，未配置到任何目标时不再兜底执行。

### Added

- 新增管理员命令 `/推文记录清理 确认`，可清理全部分组的 seen 索引并删除旧版 KV seen；也支持 `/推文记录清理 分组名 确认` 只清指定分组的 SQLite seen。
- 新增管理员命令 `/订阅删除 账号1,账号2 [分组名]`，可从全局或指定分组删除订阅账号。

### Changed

- 订阅管理命令统一为 `/订阅列表`、`/订阅导入`、`/订阅删除`、`/订阅导出`、`/订阅去重`；不再注册 `/推文订阅列表`、`/推文订阅去重`、`/博主导出`。
- 暂存队列状态文案从“暂存博主”调整为“暂存账号”。

## [0.9.0] - 2026-06-09

### Added

- 新增即时普通推送流程：后台定时检查发现新推文后，普通目标会在单条推文完成翻译、媒体下载、AI 识图和 AI 评论后立即发送；QQ/`aiocqhttp` 合并转发目标仍按阈值缓冲到本轮最后统一发送。
- 手动 `/推文` 普通发送也改为单条处理完即发，并在单条标题中显示“本次结果 x/总数”。
- `/推文检查` 改为当前会话作用域：不带分组名时按当前会话 UMO 反查唯一已启用分组；带分组名时也必须当前会话属于该分组 `push_targets` 才执行。
- `/推文检查` 在开启暂存发布的分组中会临时绕过暂存队列，只向当前会话即时发送本次新推文；结果末尾追加当前分组暂存状态、待发布数量、失败重试数量、暂存博主、下次发布时间，并提示 `/推文队列`、`/推文发布`。
- 定时推送、暂存发布和即时推送的推文正文新增来源分组显示，便于同一目标会话同时属于多个分组时区分消息来源。
- AstrBot 设置界面改为按“基础、媒体、AI 翻译、AI 评论、AI 识图、定时检查、暂存发布、推送目标与分组、并发与限流”分组展示；保留旧扁平配置兼容迁移。
- 新增单条 AI 处理摘要日志，把翻译、媒体、识图、评论结果合并为一条 `AI processed` 日志；批量处理和推送日志增加 `progress=x/总数`。

### Changed

- `/推文` 和 `/镜像测试` 的开始提示从“最近 N 条推文”改为“最近最多 N 条推文”，避免与实际返回数量或单条即时发送标题混淆。
- 移除所有命令别名，只保留 README 中列出的主命令入口。
- `/镜像测试` 的镜像站参数改为必须填写完整 `http://` 或 `https://` 地址，不再接受裸域名。
- AI 评论现在必须在翻译或识图实际产生结果后才会触发；翻译和识图都跳过时不再仅凭原文生成评论。
- AI 识图结果不再单独显示在推文消息中，只作为 AI 评论上下文使用。
- 单条即时处理时，翻译、识图、评论的普通成功/跳过日志收敛为一条摘要；媒体清理纯成功日志降为 debug，失败仍保留 warning。
- 飞书/Lark 手动即时发送的 post 标题会使用“本次结果 x/总数”等手动标题覆盖。

### Fixed

- 修复即时普通推送绕过 `send_user_interval` 的问题；第二个及后续即时 batch 发送前会按配置等待。
- 修复推送统计按 `username + new_count` 合并导致不同批次误合并的问题，统计改为按内部 batch key 合并。
- 修复即时发送异常或取消时媒体没有可靠清理的问题；没有缓冲目标时发送结束或异常都会清理媒体，有缓冲 QQ 目标时异常路径也会清理当前 batch。
- 修复手动 `/推文检查` 在暂存分组中把当前会话检查产生的新推文写入全局 pending queue，导致后续 `/推文发布` 或定时暂存发布发到原配置目标的问题。
- 修复 AI 全关时摘要日志显示 `vision=unknown`、`comment=unknown` 的问题，现在显示 `off`。
- 修复无 `status_id` 的单条即时推文在全局进度下无法匹配翻译报告，摘要显示 `translation=unknown` 的问题。
- 修复飞书/Lark 手动即时发送卡片标题仍显示“最近 1 条推文”的问题。

## [0.8.0] - 2026-06-08

### Added

- 新增 SQLite 存储后端，替代 KV 存储用于分组配置和已见推文 ID 跟踪，支持 500+ 博主订阅。
- `storage_backend` 保留为兼容配置项，但运行期固定使用 `sqlite`；旧 KV 仅作为 seen 数据启动迁移来源。
- 新增 `/订阅导入` 管理命令，账号之间只用英文逗号分隔；不带分组时写入全局 `watch_users`，也可在账号列表后用空格追加分组名称、分组 ID 或别名写入对应自定义分组；只接受 `用户名` 或 `@用户名`，单次最多 50 个。
- 数据库文件存放在 AstrBot 插件数据目录：`data/nitter_tweets.db`。
- SQLite 表结构：`meta`（schema version、迁移标记）、`groups`（分组配置）、`group_users`（订阅账号）、`group_targets`（推送目标）、`seen_tweets`（已见推文）、`pending_tweets`（待推队列）、`pending_media`（暂存媒体）。
- 自动从 KV 存储迁移已见推文数据到 SQLite，迁移使用事务保证原子性，失败可重试。
- 配置分组自动同步到数据库，使用指纹追踪避免重复更新。
- 新增 `.gitignore` 规则，忽略 `*.db`、`*.db-wal`、`*.db-shm` 文件。
- 新增管理员命令 `/推文缓存清理`，用于一键清理普通图片/视频缓存文件，并保留暂存队列媒体目录。
- 新增暂存定时发布功能：可按分组把检查到的新推文写入 SQLite 队列，到配置时间统一发布。
- 新增管理员命令 `/推文队列` 和 `/推文发布`，用于查看暂存队列和手动立即发布。
- 新增 `cache/staged/` 暂存媒体目录，暂存入队时可预下载图片/视频，发布成功后自动删除。

### Changed

- 存储层升级为主版本变更（0.7.0 → 0.8.0）。
- 已见推文 ID 按 `group_id + username` 独立存储和查询，不再整组读写。
- 每个账号只维护最近 300 条 seen ID（`SEEN_LIMIT_PER_USER`），自动清理旧记录。
- 取消订阅的账号不会立即删除 seen 记录；超过 30 天仍未重新订阅的孤儿 seen 记录会在配置同步时清理。
- `/推文状态` 改为先显示全部分组项合计，再分别显示默认分组和自定义分组详情，避免把 `global` 分组误读为总数。
- `/推文状态` 和 `/推文检查` 的账号、目标等长列表最多显示 10 项，并追加“还有 N 个”提示。
- SQLite 连接允许跨 `asyncio.to_thread()` 线程串行访问，避免 AstrBot 调度器启动和 `/推文状态` 查询时触发 `SQLite objects created in a thread can only be used in that same thread`。
- 调度器初始化时执行迁移和配置同步，失败时记录错误并在 5 分钟后重试。
- 存储运行后端固定为 SQLite；即使旧配置仍写着 `kv_legacy`，也会改用 SQLite 并只从旧 KV 导入 seen 数据。
- `pending_tweets` 从占位表升级为实际发送队列，并新增 `pending_media` 记录暂存媒体索引和路径。

### Fixed

- 修复 SQLite 线程安全问题，使用 asyncio 锁保护数据库连接。
- 修复 SQL DELETE 查询逻辑错误，正确保留最新的 N 条记录。
- 修复配置指纹碰撞风险，从 64 位增加到 128 位。
- 修复资源泄漏，调度器停止时正确关闭数据库连接。
- 修复事务失效问题，移除 autocommit 模式以支持回滚。
- 配置关闭视频/GIF 附件时，跳过视频的日志从 warning 降为 info。
- `media_cache_retention_days=0` 改为本轮发送流程结束后立即删除已下载媒体文件。

### Technical

- 所有 SQLite 操作通过 `asyncio.to_thread()` 包装，避免阻塞事件循环。
- 数据库初始化时执行完整性检查（`PRAGMA integrity_check`）。
- 旧 KV 数据永不自动删除，仅作为 SQLite 启动迁移来源保留。

## [0.7.0] - 2026-06-08

### Added

- 新增插件配置里的 `tweet_groups` 自定义推文分组列表，可在 AstrBot 后台手动配置每个分组的关注账号、推送目标、间隔检查、每日定点、拉取条数和发送间隔。
- `/推文状态` 现在会显示全局分组和自定义分组总数、启用数量，并展开每个分组的订阅账号、推送目标和定时策略。

### Changed

- 定时调度启动日志改为按所有分组统计账号数和推送目标数，不再只统计全局 `watch_users` / `push_targets`。

### Fixed

- QQ/`aiocqhttp` 合并转发在同账号一次发送较多推文时按每批最多 8 条自动分批，降低大合并包在平台侧漏节点的风险。

## [0.6.10] - 2026-06-08

### Added

- 新增定时检查运行时分组模型，现有 `watch_users`、`push_targets` 和定时配置会作为 `global` 全局分组运行，为后续自定义账号分组打基础。

## [0.6.9] - 2026-06-07

### Changed

- 飞书/Lark 不再进入合并调度；逐账号普通发送时优先使用原生 `post` 将正文和本地图片放在同一条消息中，失败时降级到原有 `text` 正文加普通媒体附件路径。
- `merge_tweet_threshold` 收敛为 QQ/`aiocqhttp` 专用合并转发阈值；飞书/Lark、Telegram、微信 OC 和其他平台不再随阈值进入合并调度，始终逐账号普通发送。
- README、配置提示和插件元数据同步说明飞书/Lark 不支持 QQ 式合并转发，飞书单账号消息仅作为平台原生展示优化。

## [0.6.0] - 2026-05-23

### Added

- 新增 `media_cache_retention_days` 配置，默认自动清理超过 3 天的媒体缓存。

### Changed

- 重写 enricher 模块，识图与评论的 provider 解析统一为单一 `_resolve_provider` 方法。
- 统一日志前缀为 `[NitterTweets]`。

## [0.5.0] - 2026-05-21

### Added

- 新增 AI 评论功能，可按概率调用文本大模型，为推文追加简短中文点评。
- 新增 AI 识图功能，可按概率调用视觉模型识别推文第一张图片，并把描述追加到推文节点。
- 新增 `comment_provider_id`、`vision_provider_id`、概率和提示词配置，默认关闭以避免升级后自动产生模型调用费用。
- 新增管理员命令 `/推文订阅列表`，用于查看当前有效订阅作者、重复项和无效项统计。

### Changed

- 定时调度默认不再在插件重启后立即检查新推文；如需旧行为可开启 `check_on_startup`。
- 手动查询和定时推送都会在媒体下载后执行 AI 增强；模型调用失败时保留原推文、链接和媒体发送。
