# 贡献流程

## 改动流程

1. 读 `AGENTS.md`。
2. 读相关专题文档和测试。
3. 小步修改，单次提交只围绕一个目的。
4. 补最小回归测试。
5. 同步用户文档、配置文档或维护文档。
6. 跑对应测试。
7. 检查 `git status --short`。

## 改动边界

- 不做无关重构。
- 不移动运行数据。
- 不提交缓存、数据库、下载产物。
- 不修改用户未要求的文案和行为。
- 不绕过已有平台适配器直接发送。
- 不在业务流程里散落配置读取，优先走已有 reader/service。

## 分支和提交

- 目标分支按维护者要求；修复功能分支时推回当前分支。
- 提交信息使用简短祈使句。
- 不写 AI attribution。
- 不写 `Generated with ...`。
- 不写 `Co-Authored-By`。

## PR 或交付说明

说明必须包含：
- 改了什么。
- 为什么改。
- 影响哪些命令、配置或平台。
- 跑了哪些测试。
- 有没有未验证项。

不要只写“修复问题”。
